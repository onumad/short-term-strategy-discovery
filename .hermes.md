# Hermes Project Instructions

Hermes is the primary agent for this repo. Keep `AGENTS.md` for portable cross-agent guidance, but treat this file as the Hermes-specific working contract.

## Repo Hygiene

- Keep edits scoped to the requested task.
- Prefer inspectable, boring code over clever abstractions.
- Avoid unrelated refactors, generated noise, and formatting churn.
- Test appropriately for the risk and blast radius before reporting completion.
- Do not commit unless explicitly asked.
- When committing, use concise Conventional Commit messages.

## Project Scope And Safety

- This is a research and simulation workspace for short-term intraday futures edge discovery.
- Do not add live trading approval.
- Do not add broker adapters, order routing, API-key storage, webhooks, or automated execution.
- Do not implement LLM-driven trade entries, exits, sizing, or discretionary decisions.
- Treat strategy results as paper-trading candidates only until independently validated.

## Environment

Use the project virtual environment:

```bash
./.venv/Scripts/python.exe
```

Install/update dependencies with:

```bash
./.venv/Scripts/python.exe -m pip install -r requirements.txt
```

## Canonical Verification

Before reporting code changes as complete, run:

```bash
./.venv/Scripts/python.exe -m unittest discover -s tests -v
```

For changes that touch data loading or audit assumptions, also run:

```bash
./.venv/Scripts/python.exe scripts/audit_data.py
```

For changes to a specific research phase, run that phase script when practical and report generated output paths.

## Research Rules

- Use only local CSV files under `data/raw` unless the user explicitly changes scope.
- Preserve the required schema: `timestamp,symbol,open,high,low,close,volume`.
- Use `America/New_York` time.
- Preserve CME-style session handling: bars at or after `18:00 ET` map to the next trading session.
- Treat RTH as `09:30-16:00 ET`.
- Preserve bar-start timestamp semantics unless a change explicitly documents otherwise.
- Avoid lookahead: indicators and signals may only use information available at the decision timestamp.
- Prefer deterministic, serializable strategy configs.
- Include costs/slippage assumptions in strategy reports.
- Document commands, assumptions, generated reports, generated CSVs, and chart/trade-log paths.

## Git Workflow

- Work directly in this repo unless the user asks for a clone/worktree.
- Check `git status --short --branch` before editing and before committing.
- Keep `.venv/`, caches, and generated Python bytecode ignored.
- Do not push unless the user explicitly asks.
